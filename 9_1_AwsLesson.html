Vim
Vimは通常のエディタとは大きく異なる操作体系を持つエディタです。
キーボードだけで高速に操作できるため、慣れるとPC作業の効率が大きく向上します。
また、サーバ環境ではGUIが使えない場面も多く、Vimはそのような環境で非常に役立ちます。

Vimの特徴

Vimには「モード」という概念があり、入力する内容によって操作が切り替わります。
この仕組みにより、マウスを使わずに高速で編集ができるようになっています。
▼ 通常のエディタとの違い
・キーボード操作が中心で、マウスをほとんど使いません。
・編集モードとコマンドモードが分かれており、操作が明確です。
・設定や拡張が豊富で、作業効率を大きく高められます。
▼ インフラで使われる理由
・サーバ環境ではGUIが使えないことが多く、Vimが標準で利用できます。
・SSH接続先でも同じ操作体系で編集できるため、環境に左右されません。
・設定ファイルの編集が高速に行えるため、作業効率が向上します。
【作業効率が上がる理由】
・キーボードから手を離さずに編集できるため、操作が速くなります。
・繰り返し作業をマクロで自動化できます。
・検索・置換・移動などが直感的かつ高速に行えます。


Ubuntu（仮想環境）でVimをインストールする手順

Ubuntuでは、以下の手順でVimをインストールします。
パッケージ処理が途中で中断されている場合があるため、最初にdpkgの再設定を行います。 

sudo dpkg --configure -a
sudo apt-get install vim

これにより、途中で止まっていたパッケージ処理が修復され、その後Vimを正常にインストールできます。

コマンドの説明

sudo dpkg --configure -a

・途中で中断されたパッケージ処理を再設定します。
・インストールや更新が途中で止まった場合に使用します。

sudo apt-get install vim

・Vimをインストールするコマンドです。
・apt-getはパッケージ管理ツールで、installはインストールを意味します。

Bashとは

Bash（Bourne Again Shell）は、Linuxで広く使われているシェルです。
シェルとは、OSに命令を伝えるためのインターフェースのことです。
コマンドを入力して操作するため、サーバ管理や自動化に欠かせない存在です。
到達目標
Vimは通常のエディタと異なり、モード切り替えによって高速に編集できる仕組みを持つことを理解する。
サーバ環境ではGUIが使えない場面が多く、Vimがインフラ作業で標準的に利用される理由を説明できるようになる。
UbuntuでのVimインストール手順として、dpkgの再設定 → apt-get install の正しい流れを実行できるようになる。
